DESIGN SYSTEM DOCUMENTATION

DESIGN PRINCIPLES

THE NON-NEGOTIABLES FOR FUTURE AGENTS

FOR FUTURE AGENTS: This mockup is the source of truth. When implementing in React, match these exact colors, spacing, and behaviors. The pixel art aesthetic is CORE to the game's identity - do not introduce rounded corners, gradients that aren't explicitly shown here, or fonts other than Press Start 2P.
1. PURE PIXEL ART COMMITMENT
We use a SINGLE font (Press Start 2P) throughout the entire UI. No mixing fonts for "visual hierarchy" - hierarchy comes from size, color, and spacing. All tiles are SQUARE with hard edges. No border-radius anywhere except the DM eye (which is literally an eye and needs to be round).
CORE PRINCIPLE
2. SELF-CONTAINED / NO EXTERNAL DEPENDENCIES
All textures, icons, and patterns are inline SVG or data URIs. No external texture URLs (they can fail, change, or slow loading). No Tailwind CDN. The mockup runs anywhere with just a browser and the Google Fonts CDN for Press Start 2P.
CORE PRINCIPLE
3. LIVING ATMOSPHERE
The dungeon is ALIVE. Floating dust particles drift upward. The mystic aura breathes. The DM eye looks around. Threat meters pulse. This creates a sense that the dungeon is watching and reacting - not just a static game board.
CORE PRINCIPLE
4. MYSTIC PURPLE AS SIGNATURE
The mystic purple (#7030b0) is our BRAND COLOR. It appears in: the DM panel border, the DM eye, active rune states, question mark tiles, hover glows on unrevealed tiles, and the breathing aura. It distinguishes us from generic dark fantasy games.
KEPT
5. TACTILE FEEDBACK
Every interactive element has three states: normal, hover (lifts up with shadow), and active/pressed (pushes down). Tiles have a 3D beveled look with light source from top-left. Buttons have visible "depth" via box-shadow that compresses on click.
KEPT
REJECTED PATTERNS: Multiple fonts (Gemini used 4), external texture URLs, rounded corners on tiles, Tailwind/utility-class CDNs, generic Material icons, glassmorphism effects, RGB/CRT subpixel tricks. These dilute the pixel art identity.

DESIGN DECISIONS LOG

WHAT WE CHOSE AND WHY

Why single font instead of a font stack?
Gemini v4 used 4 fonts (Cinzel for headings, Crimson Pro for body, JetBrains Mono for numbers, Press Start 2P for retro accents). This looked polished but felt like a web app, not a retro game. We chose Press Start 2P exclusively because: (1) It commits fully to the pixel art aesthetic, (2) It forces hierarchy through size/color/spacing instead of font changes, (3) It's more memorable and distinctive.
KEPT: Single font REJECTED: Font mixing
Why inline SVG patterns instead of external textures?
External texture URLs (like transparenttextures.com) can fail, change, slow loading, or be blocked. Inline SVG data URIs are self-contained - the mockup works offline, loads instantly, and never breaks due to external changes.
KEPT: Self-contained REJECTED: External URLs
Why no rounded corners on tiles?
Rounded corners break the pixel art aesthetic. Classic Minesweeper, retro games, and 8-bit/16-bit UI all use hard square edges. The only round element is the DM eye (which is literally an eyeball). Gemini's v4 added border-radius to tiles which looked modern but lost the retro feel.
PIXEL ART COMMITMENT
Why mystic purple as brand color?
Most dark fantasy games use gold/red/stone. Mystic purple (#7030b0) distinguishes us while fitting the "dungeon magic" theme. It appears in: DM panel, active runes, question tiles, hover states, the breathing aura. It says "this dungeon has ancient magic" not just "this is a dark cave."
BRAND IDENTITY
Why animated corner accents on panels?
Borrowed from Gemini v4 - panels have small gold corner accents that grow on hover. This adds subtle interactivity without breaking the pixel aesthetic (we kept them square, not rounded). It rewards exploration and makes the UI feel responsive.
FROM GEMINI
Why segmented health bar instead of continuous?
Segmented bars (like Diablo orbs, Dark Souls HP) make damage/healing more readable. You can count segments. Each hit visibly removes chunks. The critical state (low HP) triggers pulsing animation for urgency.
BOTH VERSIONS
Why does the DM eye look around?
The Constitution says "Dungeon Is Alive" - the DM should feel like it's watching you. The eye subtly moves on a 6s cycle (not frantic, not static). Combined with the "WATCHING" status indicator, it creates presence without being distracting.

IMPLEMENTATION NOTES

FOR FUTURE AGENTS BUILDING THE REACT VERSION

WHEN IMPLEMENTING IN REACT:
1. Create CSS custom properties file matching :root variables exactly
2. Use Tailwind ONLY if you extend it with these exact colors (no defaults)
3. Atmosphere layers should be fixed-position siblings of the game container
4. Tile states should be CSS classes, not inline styles
5. All animations should use the timing tokens (--tick, --beat, etc.)
6. Icons should be React components wrapping the exact SVG paths shown here
DO NOT:
• Use emoji for icons (❤️🛡️💰) - use the SVG icons shown here
• Add Tailwind's default colors - only use our palette
• Use external icon libraries (Heroicons, Lucide, etc.)
• Add border-radius to tiles or panels
• Change the font or add additional fonts
• Use external texture URLs
• Skip the atmosphere layers - they're essential to the feel
